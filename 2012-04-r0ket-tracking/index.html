... a quick introduction!

r0ket

Hardware

  • USB-Port (Mass-Storage, ...)
  • m0dulbus with SPI, I²C, 3.3V, etc..
  • LC-Display Nokia 1200 (96x68)
  • 32-bit ARM Cortex-M3 with 32KB flash, 8KB SRAM
  • 2.4 GHz Transceiver (OpenBeacon)
  • ~4 MiB Dataflash

OpenBeacon

OpenBeacon is based on Open Source software and a very flexible and reprogrammable lowcost Open Source RFmodule.

http://www.openbeacon.org/

Lessons learned from v1.0

  • Data consistency is crucial!
  • Extremly slow when using CSS animations
  • New features hard to implement

Building v2.0...

New approach

  • Keep data consistent
    • Track floorchanges
    • r0kets appear/disappear
    • Reader disconnect
    • Remove unused data
  • Use canvas instead of DOM
  • Draw a floormap

Demonstration

Result

  • CPU usage has gone down a lot
  • You have to render everything each frame
  • Aligning floors isn't easy
  • OOP helps (even in JS)
  • Javascript with canvas outperforms HTML + CSS animations
  • Javascript performance tweaks (prototypes instead of lambdas)
